# M1 ‚Äî `check` + `example`, published to npm

**Problem:** envpact is a CLI shell with no commands. M1 delivers the two commands that make
the pact real ‚Äî validate `.env` against a zod schema (`check`) and generate `.env.example`
from it (`example`) ‚Äî and ships `envpact@0.1.0` to npm.

## Decisions (leivaa, 2026-07-23)

- **D-101 ‚Äî Config lives under an `"envpact"` key in `package.json`** (chosen over an
  `env.schema.ts` convention or always-explicit flags):

  ```jsonc
  {
    "envpact": {
      "schema": "./env.schema.ts", // required ‚Äî no default
      "env": ".env", // optional, default shown
      "example": ".env.example", // optional, default shown
    },
  }
  ```

  Missing/invalid config exits `2` with a copy-pasteable snippet of exactly what to add.
  The config shape is itself validated with zod (dogfooding at the door).

- **D-102 ‚Äî Runtime deps: `zod` ^4 + `dotenv`** (chosen over writing our own parser or full
  zero-dep). `dotenv` is used **parse-only** (`dotenv.parse()` on file contents ‚Äî no
  `process.env` mutation, no `.config()` side effects). zod is envpact's own dependency;
  interop with the user's zod instance stays **structural** (`safeParse`, `_zod` protocol ‚Äî
  never `instanceof`), so version skew within v4 doesn't break `check`.

- **D-103 ‚Äî Schema loading is native `import()`.** Node ‚â• 24 type-strips TS, so
  `./env.schema.ts` loads with no bundler/jiti dependency. The module's **default export**
  must be a zod object schema; anything else exits `2` naming what was found and what was
  expected. Erasable-syntax-only TS (no enums/namespaces in schema files) ‚Äî documented.

- **D-104 ‚Äî Exit code `2` for usage/config errors**, distinguishing "your env broke the
  pact" (`1`, the CI signal) from "envpact is misconfigured" (`2`). README table updates
  from PR2.

- **D-105 ‚Äî `check` checks the file, not the process.** M1 validates the configured `.env`
  file only (missing file = exit `1`, the pact can't hold); merging `process.env` is a
  possible later flag, not M1.

- **D-106 ‚Äî `example` renders from schema introspection**: declaration order preserved;
  `.describe()`/`.meta({ description })` become `#` comments; zod defaults become
  pre-filled values; everything else renders `KEY=`. Only object schemas supported.

- **D-107 ‚Äî CI from the first PR**: GitHub Actions on PRs + main ‚Äî frozen install, lint,
  typecheck, test, build, **and `pnpm audit`** (codifying the workspace zero-vulnerabilities
  rule as a merge gate).

## Sub-PR breakdown

1. **PR ‚Äî CI workflow** (lead-direct: boilerplate). Actions pipeline per D-107.
2. **PR ‚Äî config + schema loading + `check`** (implementer ‚Üí reviewer loop). `config.ts`,

3. **PR ‚Äî `example` + release prep** (implementer ‚Üí reviewer loop). `commands/example.ts`,

Then: dogfood `check` + `example` against a realistic fixture mirroring the portfolio
repo's env contract, and publish to npm (pausing if `npm whoami` isn't authenticated).
